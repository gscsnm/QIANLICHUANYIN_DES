客户端主要流程：

建立登录界面
用户输入用户名
获得用户名
将用户名，ip，端口等信息传给服务器，连接服务器。
接收服务器信息
进入用户列表界面
建立线程等待服务器控制消息
			接收到消息
			判断消息类型
			if 请求通话  转到  请求通话界面
			if 拒绝通话  转到  用户列表界面
			if 新用户    转到  用户列表更新
			if 退出  转到   退出处理程序
			if 声音      加入  播放列表 等待播放

通话流程：

接受或发起通话
双方同意通话
开始通话
开启两个线程
录音线程：
	录音
	加密
	转成json发送到服务器，有服务器转发给对方
	
播放线程：
	接收网络信息流
	如果是语言流，解析jason为byte数组
	解密
	加入播放列表
	播放



服务器主要流程：

建立服务器
开始监听端口
有客户端连接
获取socket对象
建立新用户
建立线程接收此用户消息


	线程运行流程：

获取此用户通讯流对象in  & out
等待接收消息
收到消息
判断消息类别
if 文字消息或对话请求或接受对话请求或拒绝用户对话请求的消息或语音信息  then   直接发送给对方
if 用户登录消息   then   
			更新服务器上的用户信息
			发送用户列表给此用户
			发送新用户信息给所有在线用户
